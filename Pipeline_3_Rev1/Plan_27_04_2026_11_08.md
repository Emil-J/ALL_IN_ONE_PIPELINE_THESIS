1. **SAVE_FLIGHT_DATA** — per-frame query frame JPEG + raw IMU row JSON saved alongside run output
2. **SAVE_ANALYSIS_DATA** — PX4 GPS_INPUT CSV (MAVLink MSG 232) + analysis extras per run

## Research Summary

### What results.csv (31 cols) already provides — no flag changes needed

These columns cover most analysis without additional flags:
- Errors: `final_lat/lon` vs `gps_lat/gps_lon` → online EKF error via haversine
- Homo errors: `homo_lat/lon` and `homo_corrected_lat/lon` vs `gps_lat/gps_lon`

- Turns: `roll_deg` (bank angle)

- **Per-component timing**: not yet instrumented in temporal_searcher.py
- **Frame JPEGs + IMU JSON**: not currently saved during runs
- **PX4 GPS_INPUT CSV**: not currently generated

### Step 2 — `src/temporal_searcher.py`

Add per-component timing to `_process_frame_N()` (temporal_tracking path) and `_process_frame_0()` (cold_start).

**How**: bracket each component with `t = time.perf_counter()` before and compute `(time.perf_counter() - t) * 1000` after. Only collect the dict when `config.SAVE_TIMING_DATA` is True (avoid even the float arithmetic otherwise).

**Components to time in `_process_frame_N()`:**
- `pf_predict_ms`: the `pf.predict(dt, ...)` call
- `first_pass_ms`: `find_tiles_within_radius()` + the SuperPoint/LightGlue first-pass candidate loop
- `meta_build_ms`: `MetaTileBuilder` assembly + second-pass neighbour expansion
- `semantic_ms`: semantic model inference on the query frame (if called)
- `homography_ms`: dual homography computation
- `pf_update_ms`: `pf.update(...)` + resample

**Components to time in `_process_frame_0()` (cold_start):**
- `cold_search_ms`: `BestFirstSearcher.search()`
- `semantic_ms`: semantic inference
- `homography_ms`: homography

**Return dict change** — add `"timing"` key:

```python
# added only when config.SAVE_TIMING_DATA
"timing": {
    "pf_predict_ms": ...,
    "first_pass_ms": ...,
    "meta_build_ms": ...,
    "semantic_ms": ...,
    "homography_ms": ...,
    "pf_update_ms": ...,
    "total_ms": elapsed * 1000,
}
```

### Step 3 — `runtime/run_pipeline.py`

#### 3a. SAVE_QUERY_FRAMES and SAVE_IMU_ROWS

Both flags write to the same `flight_data/` sub-dir under the run directory. Directory is created lazily on first write.

```python
if config.SAVE_QUERY_FRAMES or config.SAVE_IMU_ROWS:
    flight_dir = run_dir / "flight_data"
    flight_dir.mkdir(exist_ok=True)

if config.SAVE_QUERY_FRAMES:
    frame_jpg = flight_dir / f"frame_{frame_idx:04d}.jpg"
    cv2.imwrite(str(frame_jpg), cv2.cvtColor(query_frame, cv2.COLOR_RGB2BGR))

if config.SAVE_IMU_ROWS:
    imu_json = flight_dir / f"frame_{frame_idx:04d}_imu.json"
    imu_json.write_text(json.dumps(
        {k: (None if isinstance(v, float) and math.isnan(v) else v)
         for k, v in row_dict.items()}
    ))
```

`run_dir` is already defined at the start of both functions as `config.RUNS_OUTPUT_DIR / run_id`.

#### 3b. SAVE_ANALYSIS_DATA

Open two CSV writers at run start when `SAVE_ANALYSIS_DATA=True`:

1. **`px4_gps_input.csv`** — one row per frame:

```
time_usec, gps_id, ignore_flags, time_week_ms, time_week, fix_type,
lat, lon, alt, hdop, vdop, vn, ve, vd, speed_accuracy,
horiz_accuracy, vert_accuracy, satellites_visible, yaw
```

Write one row per frame using values from the EKF state (available after `_process_one_frame` updates the EKF).

2. **`analysis_extras.csv`** — extras not in results.csv:

```
frame_idx, timestamp, n_eff, particle_spread, homo_offset_north_m, homo_offset_east_m
```

`homo_offset_north_m` = `(homo_corrected_lat - gps_lat) * 111320` (available in both modes since gps_lat is in results).
`n_eff` and `particle_spread` come from `result.get("n_eff")` and `result.get("particle_spread")`.

Close both CSV files at end of run (in `finally` block).

#### 3c. SAVE_TIMING_DATA

Collect `result.get("timing", {})` for each frame into a list. At end of run:

```python
if config.SAVE_TIMING_DATA and timing_rows:
    timing_df = pd.DataFrame(timing_rows)  # frame_idx + component columns
    timing_df.to_csv(run_dir / "timing_data.csv", index=False)
```

Also record `frame_capture_ts` and `gps_estimate_ts` (time.perf_counter()) per frame:
- `frame_capture_ts`: immediately after `get_latest_frame()` returns a new frame (SimConnect) or after `load_image()` (file mode)
- `gps_estimate_ts`: after `ekf.update_position()` or after `_process_one_frame()` completes

Include these two columns in `timing_data.csv` so latency can be computed as:
`inference_latency_ms = (gps_estimate_ts - frame_capture_ts) * 1000`
