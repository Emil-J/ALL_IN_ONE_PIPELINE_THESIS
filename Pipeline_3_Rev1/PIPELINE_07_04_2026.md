## 1. Project Overview

**Goal**: GPS-free drone localization using:
- **SuperPoint + LightGlue** visual feature matching (primary position estimator)
- **Semantic terrain segmentation** (UNet++ EfficientNet-B3, secondary tile pre-filter)
- **Error-State Extended Kalman Filter** (10D, IMU dead reckoning + visual fusion)
- **Particle filter** (search region guidance)

**Domain**: Microsoft Flight Simulator 2020 drone footage matched against TMS zoom-16 orthophoto tiles from Vejle, Denmark (~55.7°N, 9.5°E).

**Challenge**: Significant domain mismatch — MSFS renders 3D perspective views while reference tiles are 2D orthophotos. This makes pure visual localization unreliable, requiring sensor fusion with IMU dead reckoning and quality gating.

### High-Level Processing Loop

```

│                         airspeed updates                            │
│                                                                     │
│  2. Visual Search      TemporalSearcher.process_frame()            │
│     ├─ Frame 0: BestFirstSearcher (exhaustive 500m radius)         │
│     └─ Frame N: ParticleFilter predict → MetaTileBuilder search    │
│         ├─ Semantic pre-filter (histogram → top-10 tiles)          │
│         ├─ SuperPoint+LightGlue matching (cached query features)   │
│         ├─ Two-pass search (first pass + 8-neighbor expansion)     │
│         ├─ Meta-tile build + verification (≥25 keypoints)          │
│         ├─ Dual homography (DLT + MAGSAC++)                        │
│         ├─ 5 measurement methods (nadir_corrected priority)        │
│         └─ Semantic confirmation (prediction tile histograms)      │
│                                                                     │
│  3. Quality Gate       CShape > 0.3 AND inliers > 20              │
│                                                                     │
│  4. Camera Correction  Shift 110m backward along heading           │
│                                                                     │
│  5. Adaptive R         R_HIGH=30² (CShape>0.5, inliers>100)       │
│                        R_MED=60² (CShape>0.3, inliers>20)         │
│                        ×2 if meta-tile unverified                  │
│                        ×3 if turning (roll > 15°)                  │
│                        ×(2.0 - 1.5×sem_conf) semantic modulation   │
│                                                                     │
│  6. EKF Update         ekf.update_position(corrected_lat/lon, R)   │
│                                                                     │
│  7. Final Position     ekf.get_state() → corrected lat/lon         │
└─────────────────────────────────────────────────────────────────────┘
```

### Two-Path Image Preprocessing

The pipeline uses **different preprocessing** for feature matching vs. semantic segmentation:

| Path | Input | Processing | Output |
|------|-------|-----------|--------|
| **Feature matching** | 1920×1079 raw frame | Heading rotation → cap to 1280px long edge | ~1280×720 |
| **Semantic model** | 1920×1079 raw frame | Resize 512×288 → center-pad 512×512 (112px top/bottom) | 512×512 |

This split is critical — the reference notebook (`feature_matching_comparison_v2.ipynb`) proved that SuperPoint+LightGlue needs full-resolution inputs. Resizing to 512×512 destroyed feature matching quality (Bug 2).

## 3. Data Pipeline

### IMU Data Flow

The raw CSV contains MSFS SimConnect telemetry. Key preprocessing:

1. **Unit conversion**: MSFS reports `_DEGREES_` variables in **radians** (Python SimConnect quirk)
2. **Axis mapping**: MSFS body frame (X=right, Y=up, Z=forward, left-handed) → NED body frame (X=forward, Y=right, Z=down):
   ```
   accel_body = [accel_z_msfs, accel_x_msfs, -accel_y_msfs] + g_body
   omega_meas = [gyro_z_msfs, gyro_x_msfs, gyro_y_msfs]
   ```
3. **Gravity synthesis**: MSFS accelerometers report coordinate acceleration (no gravity):
   ```
   g_body = [-g·sin(pitch), g·sin(bank)·cos(pitch), g·cos(bank)·cos(pitch)]
   ```
4. **EKF warmup**: Process all rows up to `START_ROW` before entering the closed loop, giving the EKF time to converge orientation/wind estimates.

### TMS Tile Coordinate System

Reference tiles use the **TMS (Tile Map Service)** Y-axis convention where Y increases southward (opposite to OSM):

```
y_tms = (2^zoom - 1) - y_osm     # integer tile index
y_tms = 2^zoom - y_osm            # continuous (fractional) coordinates
```

This distinction (Bug 5) was critical — using `(n-1)` for fractional coordinates caused a ~345m systematic offset.

## 4. Source File Reference

### `src/ekf_ins.py` — Error-State Extended Kalman Filter

**Origin**: Adapted from `MSFS2020_IMU_Pipeline/ekf_ins.py` reference implementation.

**State vector (10D error-state):**
```
δx = [δθ_x, δθ_y, δθ_z,    # orientation error (rad)
      δb_x, δb_y, δb_z,    # gyro bias error (rad/s)
      δw_n, δw_e,           # wind error NE (m/s)
      δp_n, δp_e]           # position error NE (m) — added Phase C
```

**Key methods:**
- `predict(omega, accel, dt)` — Propagate attitude via quaternion integration, position via velocity + heading
- `update_accel_mag(accel, heading)` — Roll/pitch from gravity + yaw from magnetometer
- `update_barometer(alt, ts)` — Altitude correction
- `update_airspeed(airspeed, heading)` — TAS + wind estimation
- `update_position(lat, lon, R_pos_m2)` — **Phase C**: Visual position Kalman update on states [8:10]
- `step_ekf(ekf, row, prev_ts)` — Single-row processing helper for online mode
- `preprocess_imu_csv(csv_path)` — Batch EKF run producing dead-reckoned positions

**Phase C expansion (8D → 10D):**
- P: 8×8 → 10×10 (position uncertainty initialized to 200² m²)
- F: 8×8 → 10×10 (position states propagated with identity)
- All H matrices: 8 → 10 columns
- `update_position()`: Standard Kalman update converting lat/lon to NED innovation

### `src/temporal_searcher.py` — Frame Processor Orchestrator

**Class `TemporalSearcher`** — Routes each frame to cold-start or temporal tracking.

**Frame 0 (`_process_frame_0`):**
1. Rotate query image by EKF heading for north-aligned matching
2. `BestFirstSearcher.search()` — exhaustive match against all tiles in 500m radius
3. Dual homography (DLT + MAGSAC++) on best tile
4. Quality gate → initialize particle filter with confidence-based spread

**Frame N (`_process_frame_N`):**
1. Particle filter predict (IMU motion model)
2. Get search region (3σ confidence ellipse)
3. Query rotation + semantic preprocessing
4. `MetaTileBuilder.run()` — two-pass search + meta-tile build + verification
5. Dual homography → 5 measurement methods (cascade selection)
6. Particle filter update + resample
7. Semantic confirmation (prediction tiles from disk)

**Measurement cascade (priority order):**
```python
["nadir_corrected", "trimmed_centroid", "inlier_centroid",
 "weighted_centroid", "projected_center"]
```
First valid measurement is used.

### `src/geometric_matcher.py` — SuperPoint + LightGlue Wrapper

**Class `SuperPointLightGlueMatcher`** — Feature extraction and matching.

**Feature caching (Phase 3 optimization):**
```python
# Extract once per query frame
query_feats = matcher.extract_features(rotated_query)

# Reuse for ALL tile matches (6-9 tiles per frame)
for tile in candidates:
    result = matcher.match_precomputed(query_feats, tile_image)
```

This avoids redundant SuperPoint extraction on the query (~30ms saved × 7 tiles = ~210ms/frame).

**Methods:**
- `extract_features(img)` → `{keypoints, descriptors, ...}` — SuperPoint extraction
- `match_precomputed(feats0, img2)` → match dict — LightGlue with cached query features
- `match(img1, img2)` → match dict — Full pipeline (extract both + match)

### `src/meta_tile_builder.py` — Two-Pass Search + Meta-Tile Construction

**Class `MetaTileBuilder`** — Builds composite reference images from top-matching tiles.

**`run()` orchestration:**
1. **First pass**: Match query against all tiles in search radius
   - Semantic pre-filter: If enabled and >10 candidates, score by histogram intersection, keep top-K=10
   - SuperPoint+LightGlue match each tile → rank by match count
2. **Second pass**: Take top-1 tile → match its 8 neighbors → merge rankings
3. **Build aerial meta-tile**: Stitch top-3 tiles into north-up grid (512×N canvas)
4. **Build prediction meta-tile**: Load pre-computed prediction PNGs for same tiles (no model inference)
5. **Verify**: Match query against full meta-tile → verified if ≥25 keypoints matched
6. **Save**: Timestamped PNG to `outputs/`

**Semantic pre-filter (Phase 4):**
```python
if cfg.SEMANTIC_PREFILTER_ENABLED and query_semantic_map is not None and len(candidates) > 10:
    scored = SemanticTileScorer.score_tiles(query_semantic_map, candidates, tile_loader)
    candidates = scored[:cfg.SEMANTIC_PREFILTER_TOP_K]  # Keep top-10
```

This reduces SuperPoint matches from ~24 tiles to ~10, saving ~50% search time.

**Prediction meta-tile (`build_prediction_meta_tile`):**
- Loads color-coded prediction PNGs from `REFERENCE_PRED_DIR` for the same tiles
- Stitches into same grid layout as aerial meta-tile
- Returns `None` if no prediction files exist
- Used by `SemanticConfirmer` to skip UNet++ re-segmentation

### `src/visual_measurement.py` — Rotation, Dual Homography, Measurements

**Heading rotation (`rotate_image`):**
- Rotates query frame by -heading° (CCW) so it's approximately north-aligned
- Improves feature matching since reference tiles are north-up orthophotos
- Returns affine transform matrix for coordinate back-projection

**Dual homography (`compute_dual_homography`):**
Two independent homography estimations for robustness:
- **Branch A (DLT)**: Direct Linear Transform on all correspondences — fast but outlier-sensitive
- **Branch B (MAGSAC++)**: Maximum Margin with Adaptive Confidence — robust outlier rejection
- **Winner selection**: `score = num_inliers × CShape × (1.5 if convex else 1.0)`
- MAGSAC wins ties (robustness preferred)

**CShape confidence metric:**

Adapted from the **FVL-SAR** paper (Feature-based Visual Localization for SAR):
```
CShape = 0.6 × min(terms) + 0.4 × mean(terms)
```
Where `terms` measures how "rectangle-like" the projected query quadrilateral is:
- `opposing_side`: Ratio of opposing edge lengths (min/max for each pair)
- `width_height`: Aspect ratio penalty (wide/tall)  
- `right_angle`: Deviation from 90° at corners
- `area`: Projected area vs. expected area ratio

CShape ∈ [0, 1] where 1.0 = perfect rectangle (ideal homography).  
**Quality gate threshold**: CShape > 0.3 AND inliers > 20.

**5 measurement methods (`extract_visual_measurements`):**

| Method | Description | When Used |
|--------|-------------|-----------|
| `nadir_corrected` | Query center shifted by pitch/roll before H projection | Default (priority 1) |
| `trimmed_centroid` | Mean of inlier ref keypoints, 10% outliers dropped | Fallback 2 |
| `inlier_centroid` | Mean of all inlier reference keypoints | Fallback 3 |
| `weighted_centroid` | Inlier mean weighted by inverse reprojection error | Fallback 4 |
| `projected_center` | Raw query center through homography H | Last resort |

**Nadir correction** compensates for camera tilt:
```python
shift_x = -f_px × tan(roll)
shift_y = -f_px × tan(pitch)
f_px ≈ width / (2 × tan(35°))  # approximate focal length
```

### `src/semantic_tile_scorer.py` — Histogram-Based Tile Scoring

**Class `SemanticTileScorer`** — Ranks candidate tiles by terrain similarity.

**Algorithm:**
1. Convert query semantic mask (uint8 class indices) → normalized class histogram (6 bins)
2. For each candidate tile: load prediction PNG → decode RGB → class mask → histogram
3. Score = histogram intersection: `Σ_c min(h_query[c], h_ref[c])`
4. Sort descending → top-K tiles proceed to SuperPoint matching

**RGB → class decoding:**
- 6 classes: waterbodies(0), forest_trees(1), land(2), railway(3), roads(4), buildings(5)
- Exact RGB lookup table from model's color map
- Vectorized nearest-neighbor fallback for JPEG compression artifacts (L1 distance)

**Why histogram intersection?**
- Viewpoint-invariant: oblique MSFS 3D renders and orthophoto 2D share the same terrain class distribution
- Fast: ~1ms per tile vs ~60ms for SuperPoint extraction
- Effective pre-filter: reduces 24 candidates to 10 without losing the correct tile

### `src/semantic_confirmer.py` — Semantic Double-Confirmation

**Class `SemanticConfirmer`** — Validates visual match with terrain semantics.

**Two paths:**
1. **Fast path** (prediction meta-tile available): Direct histogram intersection on pre-computed prediction tiles (~0ms model inference)
2. **Slow path** (fallback): Segment aerial meta-tile with UNet++ model (~20ms), then histogram

**Usage in pipeline:**
- Computes `sem_conf` ∈ [0, 1] (histogram intersection score)
- Modulates adaptive measurement noise R in notebook:
  ```python
  r_used *= max(0.5, 2.0 - 1.5 * sem_conf)
  ```
- Does NOT gate the final position decision (SP+LG quality gate does that)

**Analysis finding (Cell 19):** Spearman correlation with error: CShape ρ=-0.344 (significant), Semantic ρ=-0.006 (not significant). SP+LG is the primary accuracy driver; semantic supports via tile pre-filtering.

### `src/particle_filter.py` — Particle Filter for Search Guidance

**Class `ParticleFilter`** — 100-particle filter for search region estimation.

**Motion model (`predict`):**
```python
heading += gyro_z_dps × dt + N(0, σ_heading)
x += velocity × sin(heading) × dt / tile_size + N(0, σ_position)
y -= velocity × cos(heading) × dt / tile_size + N(0, σ_position)
```

**Measurement update (`update`):**
- Spatial likelihood: Gaussian on distance from particle to measurement position
- Heading likelihood: von Mises-like on heading difference
- Combined: `w *= score × spatial_lk × heading_lk`

**Resampling**: Low-variance systematic resampling when N_eff < 50, with jitter (~10m, 2°) to prevent collapse.

**Divergence detection**: Position std > 500m OR max weight < 0.01 → trigger cold restart.

**Role change (Phase C):** The particle filter is retained for **search region guidance only**. The EKF is the primary position estimator. Visual measurements update the EKF, not the particle filter's position estimate.

### `src/position_estimator.py` — Homography → GPS Conversion

**Pipeline:**
1. RANSAC homography estimation (query keypoints → reference keypoints)
2. Project query image center through H → pixel coordinates in meta-tile
3. Convert meta-tile pixel → fractional tile coordinates:
   ```
   tx_frac = x_min + px_x / 512
   ty_frac = (y_max + 1) - px_y / 512
   ```
4. Convert fractional TMS tile → lat/lon via `tile_to_latlon()`

### `src/tile_utils.py` — TMS Tile Math + Tile Loader

**Key functions:**
- `latlon_to_tile(lat, lon, zoom)` → integer TMS tile indices
- `latlon_to_tile_float(lat, lon, zoom)` → fractional TMS coordinates (sub-tile precision)
- `tile_to_latlon(tx, ty, zoom)` → SW corner of tile
- `tile_bounds(tx, ty, zoom)` → full bounds with center
- `haversine_distance(lat1, lon1, lat2, lon2)` → meters
- `find_tiles_within_radius(lat, lon, radius_m, zoom, x_range, y_range)` → candidate list

**Class `TileLoader`:**
- Loads aerial and prediction tiles from TMS directory structure
- Path: `{base}/{zoom}/{x}/{y}.png`
- Caches existence checks for fast lookups

### `src/best_first_search.py` — Cold-Start Exhaustive Search

**Class `BestFirstSearcher`** — Frame 0 initialization.

Matches query against ALL tiles within 500m of EKF position. Returns ranked list by match count. The top tile's homography provides the initial position estimate for particle filter initialization.

### `src/semantic_model.py` — UNet++ Semantic Segmentation

## 5. Improvement Phases

### Phase 1: EKF as Tile-Search Center

**Problem**: The particle filter's weighted mean was used as the tile search center. When the PF diverged, the search center drifted far from the drone, causing cascading failures.

**Solution**: Use the EKF dead-reckoned position as the search center instead. The EKF is more stable than the PF because it only drifts slowly from IMU integration error.

**Implementation**: In `temporal_searcher.py`, `_process_frame_N()` uses `imu_data['lat']`, `imu_data['lon']` (EKF estimates) as the MetaTileBuilder search center instead of `pf_estimate`.

**Impact**: Eliminated cascading search failures from PF divergence.

### Phase 2: Nadir Gate Fix + Turn R-Inflation

**Problem 1**: The nadir correction gate was backwards — it gated OUT good frames (low pitch/roll = good match conditions) and let bad frames through.

**Fix**: Inverted the condition so nadir correction is applied when pitch/roll are small (near-nadir).

**Problem 2**: During banked turns (roll > 15°), visual matches degrade due to perspective distortion, but the EKF trusted them equally.

**Fix**: Added `TURN_R_MULTIPLIER = 3.0` — measurement noise R is tripled during turns, reducing the EKF's trust in those visual measurements.

**Implementation**: Notebook Cell 4 checks `abs(bank_rad) > TURN_ROLL_THRESHOLD_RAD` and applies `r_used *= TURN_R_MULTIPLIER`.

**Impact**: Heavy turn error: 51.8m → 40.0m (23% improvement).

### Phase 3: SuperPoint Feature Caching

**Problem**: For each tile match, the query frame's SuperPoint features were re-extracted. With 7-9 tiles per frame, this meant 7-9 redundant extractions (~30ms each = 210ms wasted).

**Solution**: Extract query features once via `matcher.extract_features(query)`, then reuse via `matcher.match_precomputed(cached_feats, tile)` for all tile matches.

**Implementation**:
- `geometric_matcher.py`: Added `extract_features()` and `match_precomputed()` methods
- `meta_tile_builder.py`: `first_pass()` and `second_pass()` accept `query_feats` parameter
- `temporal_searcher.py`: Extracts features once, passes to all downstream matchers

**Origin**: Standard practice in large-scale visual localization (e.g., COLMAP, hloc). The insight is that SuperPoint extraction depends only on the query image, not the reference.

**Impact**: ~210ms saved per frame → speed improved from ~2.8s to ~2.04s/frame.

### Phase 4: Semantic Histogram Overhaul (4a–4e)

**Phase 4a: New `semantic_tile_scorer.py`**

Created a dedicated histogram-based tile scorer using class distribution comparison. The key insight: even with domain mismatch (3D perspective vs 2D orthophoto), the terrain class distribution (% forest, % roads, etc.) remains similar. Histogram intersection captures this viewpoint-invariant similarity.

**Origin**: Histogram intersection for scene recognition from Swain & Ballard (1991). Applied here because centroid-based matching was too rigid for the domain gap.

**Phase 4b: Semantic Pre-Filter in `meta_tile_builder.py`**

Before running SuperPoint on all ~24 candidate tiles, the semantic pre-filter scores them by histogram intersection and keeps only the top 10. This cuts SuperPoint matching volume by ~58%.

**Phase 4c: Histogram-Based `semantic_confirmer.py`**

Replaced centroid-based confirmation with histogram intersection. The confirmer now also accepts pre-computed prediction meta-tiles (built from disk) to skip UNet++ re-segmentation of the aerial meta-tile.

**Phase 4d: Reordered Semantic Computation in `temporal_searcher.py`**

Moved semantic preprocessing earlier in the pipeline so the semantic map is available for pre-filtering before SuperPoint matching begins.

**Phase 4e: Config Additions**

Added `SEMANTIC_PREFILTER_ENABLED = True`, `SEMANTIC_PREFILTER_TOP_K = 10` to `config/config.py`.

**Combined Phase 4 Impact**: ~50% reduction in tiles matched per frame (24 → 10), contributing to the overall speed improvement.

### Phase 5: Prediction-Based Semantic Confirmer (This Session)

**Problem**: The `SemanticConfirmer.confirm()` method was re-segmenting the aerial meta-tile using UNet++ model inference (~20ms) every frame. But pre-computed prediction tiles already exist on disk at `REFERENCE_PRED_DIR` — they're higher quality (offline model with more context) and free to load.

**Solution**:
1. `MetaTileBuilder.build_prediction_meta_tile()` — new method that loads prediction PNGs for the same tiles used in the aerial meta-tile and stitches them into a matching grid
2. `SemanticConfirmer.confirm()` — accepts optional `prediction_meta_tile` parameter. When provided, uses direct histogram intersection on the prediction tile RGB (no model inference needed)
3. `TemporalSearcher` — passes `prediction_meta_tile` from `meta_result` to `confirm()`

**Effect**: Semantic confidence distribution shifted from 0.5–0.9 (UNet inference on stitched aerial) to 0.3–0.7 (prediction tiles from disk). The prediction tiles are more calibrated — they were produced by the model on individual tiles without stitching artifacts.

### Phase 6: Verified Flag R-Modulation (This Session)

**Problem**: The `meta_tile_verified` flag (meta-tile has ≥25 keypoints matching the query) was computed but never used in the EKF update path. Unverified meta-tiles indicate the homography may be unreliable.

**Solution**: In the notebook closed-loop cell, when `meta_verified = False`, inflate measurement noise R by 2×:
```python
if not meta_verified:
    r_used *= 2.0
```

**Impact**: Combined with Phase 5, mean error improved from 38.3m → 35.7m (−6.8%).

## 6. Bug History & Fixes

### Bug 1 — TMS Y-Axis Convention (Fixed 2026-03-25)

`latlon_to_tile()` returned OSM Y (small values ~20504) but tiles on disk use TMS Y (~45031). No tiles were ever found.

**Fix**: `y_tms = n - 1 - y_osm` in `latlon_to_tile()`, reversed in `tile_to_latlon()`.

### Bug 2 — Image Preprocessing Destroyed Feature Matching (Fixed 2026-03-25)

`preprocess_query_frame()` was called before SuperPoint+LightGlue, resizing 1920×1079 → 512×288 → padded 512×512 with 224px of black border. SuperPoint found keypoints in the black border.

**Fix**: Raw frames go directly to matcher. `preprocess_query_frame()` is called only for the semantic model.

### Bug 3 — Raw GPS Used Instead of EKF Dead Reckoning (Fixed 2026-03-25)

The notebook used `row['latitude']` (ground truth GPS from MSFS) instead of EKF-estimated positions. On IMU fallback, this returned 0.0m error (measuring GPS against itself).

**Fix**: Created `ekf_ins.py` with `preprocess_imu_csv()`. EKF produces `latitude_est`, `longitude_est`, `yaw_deg`.

### Bug 4 — START_ROW=0 With Drone Outside Map (Fixed 2026-03-25)

First 430 frames are outside the reference map. Cold-start found 0 tiles, causing immediate divergence.

**Fix**: Auto-detect first row where GPS is inside reference map bounds.

### Bug 5 — TMS Float Coordinate Off-by-One (Fixed 2026-03-25)

`latlon_to_tile_float()` used `(n-1) - y_osm` instead of `n - y_osm` for continuous coordinates. This shifted all tile center computations ~345m north (one full tile).

**Fix**: Use `n - y_osm` for fractional, `(n-1) - y_osm` for integer.

### Bug 6 — EKF Column Name Mismatch (Fixed 2026-03-25)

Notebook referenced `ekf_lat`/`ekf_lon` but `preprocess_imu_csv()` returns `latitude_est`/`longitude_est`. Also double-converted heading with `np.degrees(yaw_deg)` when it was already degrees.

### Bug 7 — Search Radius Too Small (Fixed 2026-03-25)

Minimum search radius was 100m. Tiles are ~332m at zoom 16. A 100m radius couldn't find the tile the particle was standing on.

**Fix**: Minimum radius → 1.5 × tile_size (~500m).

### Bug 8 — Camera Look-Ahead Offset (Fixed 2026-04-07)

Homography positions were systematically ~110m ahead of the drone in the heading direction. The MSFS camera has a fixed forward tilt.

**Diagnostic**: Offset bearing ≈ heading ± 180° (anti-aligned). Mean offset: 112m.

**Fix**: Shift homography position 110m backward along EKF heading:
```python
corr_north = -LOOKAHEAD_M * cos(heading_rad)
corr_east  = -LOOKAHEAD_M * sin(heading_rad)
```

**Impact**: Mean error 103.5m → 9.7m (91% improvement, 50-frame test).

## Appendix: MSFS-Specific Notes

### SimConnect Quirks
- Python SimConnect returns `_DEGREES_` variables in **radians** (not degrees)
- Accelerometers return coordinate acceleration in **ft/s²** (no gravity) → convert to m/s² and add synthesized gravity
- Body axis mapping: MSFS (X=right, Y=up, Z=forward, LH) → NED (X=fwd, Y=right, Z=down, RH)

### Camera Look-Ahead
The MSFS drone camera has a fixed forward tilt. The image center corresponds to ground ~110m ahead of the drone, not directly below. This creates a systematic forward bias in all homography-derived positions.

Correction applied in the notebook loop:
```python
LOOKAHEAD_M = 110.0
heading_rad = np.radians(ekf_yaw)
corr_north = -LOOKAHEAD_M * np.cos(heading_rad)
corr_east  = -LOOKAHEAD_M * np.sin(heading_rad)
homo_corrected_lat = homo_lat + corr_north / 111320
homo_corrected_lon = homo_lon + corr_east / (111320 * np.cos(np.radians(homo_lat)))
```

This is empirically tuned for the specific flight and camera configuration. Different camera tilt angles or altitudes would require recalibration.
